### 2. Unit 6 — Prompt tuning and content quality pass (not yet started)

This is the important one. Current output from `qwen3:8b-q4_K_M` has several quality issues that were visible in the Cubs columns before the session ended:

1. **Hallucination** — the model invents specific plays, scores, and even player names that aren't in the day context. Example from the Cubs Walt column: "Swanson singled home two in the seventh, Imanaga went six clean innings" — fabricated specifics.
2. **Breaking character mid-column** — Walt's column literally wrote "Wait, no—wait, no. The record is 6-6." That's the model correcting itself out loud. Rule violation (no meta-commentary).
3. **Length undershoot** — columns are running 800-1100 chars (~150-200 words), short of the 300-word target.
4. **Score invention** — Tony's column claimed a specific final score that didn't match the day context.

**Prompt fixes to try, in order:**

- **Add a hard anti-hallucination rule** — e.g., "Do not invent specific plays, player names, or statistics not explicitly listed in TODAY'S FACTS above. If you don't know a specific detail, speak generally about patterns and feelings, not fabricated specifics."
- **Strengthen TODAY'S FACTS with more real data** — the current `_build_day_context()` in `sections/columnists.py` only passes yesterday's game and current record. Pass: recent game log (last 5), team leaders, hot/cold players, biggest storyline. Source these from the same `briefing.data` dict that the other sections use. See `sections/headline.py` for examples of what's available.
- **Restate the "no meta-commentary" rule more forcefully** — the current rule "Do not break the fourth wall" is being ignored. Try a negative example: "Do not write phrases like 'wait, no' or 'let me check' or 'actually that was last week'. You are confident in everything you say."
- **Try `qwen3:14b` or `llama3.1:8b`** instead of `qwen3:8b-q4_K_M` — the 8B quantized model may simply be too small for 300-word persona columns. Swap the model name in `_try_ollama()` and test one team.
- **Raise `num_predict`** — currently 768 tokens, consider 1280 to give more headroom for length.
- **Last resort: lean on the Anthropic fallback** — raise `MIN_COLUMN_CHARS` from 250 to 1200 so Ollama output gets rejected more aggressively and Claude Sonnet handles more of the load. This costs money (~90 calls per daily build) but gives much better quality. Worth testing before committing to it.

**Acceptance bar for Unit 6:**

Read a sample of 5 teams' full trios. Ask:
- Can you identify which writer wrote which column from voice alone? (yes = prompts are distinct enough)
- Do any columns invent specific plays or player names not in the context? (if yes, fix #1 above)
- Do any columns break character or correct themselves? (if yes, fix #2 above)
- Is the length 250-500 words? (if short, fix #3/#4 above)
